## Release notes

This file is an optional, free-form scratchpad for documenting **why** each
release exists and any context that doesn't belong in `CHANGELOG.md`.

For each release, add a section like:

```markdown
## 0.2.0 - 2026-03-11

- Why we cut this release now.
- Risky changes or areas to watch.
- How we validated the changes.
- Ideas and TODOs for the next release.
```

You can skip sections for tiny releases; focus on versions where extra context
will help you (or teammates) in the future.
